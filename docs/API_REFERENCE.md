## 1. Overview

Dashboard menggunakan **Supabase Client SDK** untuk akses data langsung dari frontend. Tidak ada REST API custom untuk operasi CRUD standar — semua dilakukan melalui Supabase dengan Row Level Security (RLS) sebagai pengaman.

### 1.1 Supabase Client Initialization

```typescript
import { createClient } from '@supabase/supabase-js';

const supabase = createClient(
  import.meta.env.VITE_SUPABASE_URL,
  import.meta.env.VITE_SUPABASE_ANON_KEY
);
```

### 1.2 Common Patterns

```typescript
// SELECT with filter
const { data, error } = await supabase
  .from('produksi')
  .select('*')
  .eq('tanggal', '2026-05-25')
  .order('created_at', { ascending: false });

// INSERT with auto timestamp
const { data, error } = await supabase
  .from('work_orders')
  .insert({
    nomor_wo: 'WO-2026-0001',  // Generated by Edge Function
    tipe_maintenance: 'preventive',
    prioritas: 'high',
    mesin_id: '...',
    deskripsi_masalah: 'PM rutin mesin packing #1',
    // created_at, updated_at auto-generated by DB
  });

// UPDATE
const { data, error } = await supabase
  .from('work_orders')
  .update({ status: 'berlangsung' })
  .eq('id', woId);

// Realtime subscription
supabase
  .channel('production-realtime')
  .on('postgres_changes', {
    event: '*',
    schema: 'public',
    table: 'produksi'
  }, (payload) => {
    console.log('Change received:', payload);
  })
  .subscribe();
```

## 2. Edge Functions

### 2.1 generate-wo-number

**Endpoint:** `POST /functions/v1/generate-wo-number`

Generates auto-incrementing work order number.

```typescript
// Request
{ "prefix": "WO", "date": "2026-05-25" }

// Response
{ "nomor_wo": "WO-2026-05-0042" }
```

### 2.2 check-reorder-point

**Endpoint:** `POST /functions/v1/check-reorder-point`

Checks all spare parts below reorder point and sends alerts.

```typescript
// Request (no body needed, runs on schedule)

// Response
{
  "alerts": [
    { "part_id": "...", "nama_part": "Bearing SKF 6205", "stok": 3, "reorder_point": 5 },
    ...
  ]
}
```

### 2.3 calculate-oee

**Endpoint:** `POST /functions/v1/calculate-oee`

Calculates Overall Equipment Effectiveness for a machine/shift.

```typescript
// Request
{ "mesin_id": "...", "tanggal": "2026-05-25", "shift": "pagi" }

// Response
{
  "availability": 0.92,
  "performance": 0.88,
  "quality": 0.97,
  "oee": 0.79
}
```

### 2.4 export-report

**Endpoint:** `POST /functions/v1/export-report`

Generates PDF/Excel report for a given module and date range.

```typescript
// Request
{
  "module": "produksi",
  "date_from": "2026-05-01",
  "date_to": "2026-05-25",
  "format": "xlsx"
}

// Response
{ "url": "https://storage.supabase.co/.../report-2026-05.xlsx" }
```

## 3. RLS Policies

### 3.1 Example Policies

```sql
-- Operators can only insert their own production records
CREATE POLICY "Operators can insert produksi"
  ON produksi FOR INSERT
  WITH CHECK (created_by = auth.uid());

-- Managers can view all production records
CREATE POLICY "Managers can view all produksi"
  ON produksi FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM profiles
      WHERE id = auth.uid()
      AND role IN ('manager', 'admin', 'superadmin')
    )
  );

-- Technicians can only update WOs assigned to them
CREATE POLICY "Technicians update assigned WOs"
  ON work_orders FOR UPDATE
  USING (pic_teknisi = auth.uid());
```
